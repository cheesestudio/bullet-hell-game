# Radmin VPN 联机使用指南

## 支持人数

最大支持 **4 名玩家** 同时联机（1 主机 + 3 客户端）。

---

## 前置条件

- 安装 [Radmin VPN](https://www.radmin-vpn.com/) 并创建/加入同一网络
- 安装 [Node.js](https://nodejs.org/) (运行中继服务器需要)
- 安装 ws 依赖: `npm install`

---

## 联机步骤

### 主机操作

1. 打开 Radmin VPN，创建或加入一个网络
2. 在项目目录下运行中继服务器：`node relay.js`
   - 看到 `Relay 中继服务器已启动 ws://0.0.0.0:8007` 表示成功
3. 用浏览器打开 `bullet-hell.html`
4. 主菜单选择 **网络对战** -> **创建房间**
5. 将你的 Radmin VPN IP 地址告诉其他玩家
6. 等待其他玩家加入后，按 **Enter/空格** 开始游戏

### 客户端操作

1. 打开 Radmin VPN，加入主机所在的网络
2. 用浏览器打开 `bullet-hell.html`
3. 主菜单选择 **网络对战** -> **加入房间**
4. 输入主机的 Radmin VPN IP 地址，按 **Enter** 连接
5. 连接后按 **R** 准备就绪
6. 等待主机开始游戏

---

## 端口信息

| 端口 | 用途 | 说明 |
|------|------|------|
| `8007` | WebSocket 中继 | relay.js 监听端口，需要在 Radmin VPN 网络内可达 |

> Radmin VPN 会创建虚拟局域网，所有端口在 VPN 网络内自动互通，不需要额外的端口映射或路由器配置。

---

## 网络要求

- Radmin VPN 已正确连接
- 所有玩家在同一个 Radmin 网络
- Windows 防火墙允许 Node.js 通过端口 8007 通讯
- 不需要公网 IP，不需要端口映射，不需要路由器设置

---

## 操作按键（联机模式）

联机模式下，每位玩家在自己的电脑上使用相同的按键操作自己的角色：

| 操作 | 按键 |
|------|------|
| 移动 | 方向键 / WASD |
| 炸弹 | 空格 / B |
| 射击 | 自动射击 |

> 本地双人模式使用不同键位（P1方向键，P2 WASD），联机模式下每人各用自己的键盘。

---

## 大厅操作

| 状态 | 操作 |
|------|------|
| 选择菜单 | 上/下方向键切换，Enter/空格确认 |
| 输入IP | 键盘输入IP地址，Backspace删除，Enter连接 |
| 客户端等待 | R 键切换准备状态 |
| 主机等待 | Enter/空格 开始游戏（至少1人连接） |
| 任意状态 | ESC 返回上级 |

---

## 同步机制

- **固定时间步**: 网络模式使用 1/60s 固定帧率，确保所有客户端模拟步调一致
- **Seeded PRNG**: 所有客户端使用相同随机种子（mulberry32），Boss弹幕、粒子效果等完全一致
- **输入镜像**: 每帧广播本地输入状态（5-bit bitmask），远端玩家按接收到的输入更新
- **WebSocket 中继**: relay.js 作为无状态广播中继，转发所有消息并标记发送者ID

---

## 故障排查

| 问题 | 解决方案 |
|------|----------|
| relay.js 启动失败 | 确认已运行 `npm install`，端口 8007 未被占用 |
| 客户端连接超时 | 检查 Radmin VPN 是否在同一网络，能否 ping 通主机 IP |
| 连接后无响应 | 检查 Windows 防火墙是否允许 Node.js 网络访问 |
| 游戏中断开 | relay.js 进程是否仍在运行，VPN 连接是否稳定 |
| 游戏不同步 | 确保所有玩家使用相同版本的 bullet-hell.html |
